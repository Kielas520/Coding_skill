# 文档模板

## 目录结构

```
docs/
├── design/<模块名>/         # 设计文档（按模块拆分）
│   ├── overview.md          # 任务实现纵览
│   ├── data-flow.md         # 数据流
│   ├── interface.md         # 输入输出接口
│   ├── design-rationale.md  # 代码设计思路
│   └── pipeline.md          # 数据管线总览
├── dev-log/<日期>.md         # 开发日志（按日期）
├── todo/
│   ├── current.md           # 当前迭代待办
│   └── backlog.md           # 积压任务
├── bugs/<BUG-编号>.md        # Bug 和 Issue 记录
├── deprecated/<方案名>.md    # 废案记录
├── env-setup/
│   ├── python.md            # Python 环境配置
│   └── dependencies.md      # 系统依赖
└── startup/
    ├── debug.md             # 调试启动
    └── production.md        # 完整启动
```

## 使用说明

生成文档时：
1. 先确认文档类型和目标模块
2. 读取现有代码和已有文档，避免凭空编造
3. 按模板结构填充，未知信息标注"待补充"而不是猜测
4. 生成后列出已创建/更新的文件列表
5. 询问用户是否需要调整
